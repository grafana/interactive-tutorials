Section 1: Launch Kubernetes Monitoring

Open the Kubernetes Monitoring overview to see the health of every connected cluster.

  • Observability groups infrastructure experiences, including Kubernetes Monitoring. Click Observability in the left navigation.
  • The Kubernetes tile opens the dedicated experience with curated dashboards and drilldowns. Select Kubernetes to open the Kubernetes Monitoring overview.

Section 2: Explore clusters, nodes, pods, and containers

Follow the curated drilldowns to move from cluster health to container-level optimization data without losing context.

  • The overview tile shows how many clusters are currently reporting metrics. Click the Clusters summary tile to open the full cluster list.
  • Cluster rows surface status, workload count, and alert totals so you can pick the busiest cluster first. Open the first cluster to view its detail page.
  • Cluster information summarizes location, total capacity, and current resource usage. Review the Cluster information panel for inventory context.
  • Optimization graphs surface over/under-provisioning trends across CPU and memory. Examine Cluster optimization to spot efficiency opportunities.
  • Requests vs limits help identify workloads that risk eviction or noisy neighbors. Inspect the CPU requests & limits panel.
  • Memory requests and limits reveal pods that might crash loop because of under-provisioning. Review the Memory requests & limits panel.
  • Node tables combine utilization, alert count, and kubelet status for quick triage. Scroll to the Nodes list within the cluster.
  • Jumping into a node keeps breadcrumb navigation intact so you can bounce back easily. Click the first node name to open the node detail page.
  • Long-term CPU and memory trends reveal which nodes might need rescheduling or scaling. Review the Node optimization graphs.
  • Pod tables surface workload ownership, namespace, restart counts, and current status. Scroll to the Pods list for this node.
  • Pod detail pages inherit filters so time ranges and cluster context stay consistent. Open the first pod from the list.
  • Containers inherit workload labels, making it easy to trace back to deployments or statefulsets. From the pod detail page, open the first container.
  • Container information links back to the namespace, node, pod, and workload that own it. Inspect the Container information section.
  • Optimization hints recommend request and limit adjustments based on observed usage. Review the Container optimization insights.
  • Breadcrumb links let you pivot back to namespace-level visibility without losing context. Use the namespace link to jump back up the hierarchy when you are ready.

What you have learned

You can now traverse the full object hierarchy to isolate issues down to individual containers.

Section 3: Search and navigation shortcuts

Combine global search with in-app menus to locate any Kubernetes object in seconds.

  • The menu item returns you to the Kubernetes overview from anywhere in Grafana. Click Kubernetes to ensure the overview is in focus.
  • Global search indexes dashboards, Kubernetes objects, alerts, and more. Open Search from the main navigation.
  • Searching by namespace, workload, pod, or container name jumps you straight to the correct page. Type the name of an object you expect, such as nginx.
  • Results show object type and hierarchy so you know exactly where each link leads. Choose a search result to navigate directly to that object.
  • Dedicated landing pages (Nodes, Namespaces, Workloads) give you sortable tables and filters. Use the navigation to open the Nodes page.
  • The page navigation button exposes quick links to sibling resources without leaving the context. Expand the page navigation menu next to the Nodes title.
  • Use Alerts to review Kubernetes-specific incidents and route them to the right team. Open the Alerts view to review active issues.

What you have learned

You can now jump between overviews, search results, and alert-focused views without breaking your flow.

Section 4: Analyze namespaces and workloads

Use dedicated performance tabs to evaluate CPU, memory, and network behavior for any namespace or workload.

  • Namespace landing pages highlight saturation and cost signals across workloads. Navigate to Namespaces.
  • Each row includes status, workloads, pods, and alert counts for rapid prioritization. Open the first namespace.
  • CPU views combine requests, limits, usage, and saturation so you can right-size workloads. Select the CPU tab.
  • Tooltips explain how each metric is calculated and why it matters. Hover any information icon inside the CPU tab.
  • Memory insights correlate working set, RSS, limits, and throttling events. Open the Memory tab.
  • Network graphs expose traffic spikes, saturation, and dropped packets. Switch to the Network tab.
  • Workloads surfaces deployments, statefulsets, daemonsets, and jobs in one place. Navigate to Workloads.
  • Workload detail pages include the same performance tabs plus rollout and replica summaries. Open the first workload in the list.
  • CPU charts highlight usage versus requests so you can estimate headroom. Confirm the CPU tab for the workload.
  • Memory trends help you catch impending OOMKills before they happen. Check the Memory tab.
  • Network panels highlight latency-sensitive workloads that may need QoS adjustments. Review the Network tab for the workload.

What you have learned

You can now evaluate namespace and workload performance across CPU, memory, and network dimensions.

Section 5: Use predictive insights

Machine learning powered predictions help you understand when to scale capacity or adjust resource requests.

  • Pick a workload with meaningful historical data to train the prediction model. Open a workload from the list.
  • Longer time ranges improve forecast accuracy by giving the model more signal. Expand the time range control and select Last 7 days.
  • CPU predictions display a forecast band and expected saturation window. Click Predict CPU usage in the CPU usage panel.
  • Memory predictions highlight when to raise limits or add replicas before OOM events occur. Click Predict Mem Usage in the memory panel.
  • Prediction cards summarize trend direction, confidence interval, and next recommended action. Review the prediction summary and close it when you are finished.

What you have learned

You can apply predictive analytics to proactively plan scaling and budget decisions.
