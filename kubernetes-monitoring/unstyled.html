Explore Your Infrastructure with Kubernetes Monitoring

Grafana Cloud's Kubernetes Monitoring provides a comprehensive view of your Kubernetes infrastructure with powerful visualization and analysis tools. In this tutorial, you'll learn how to:

Overview

Use the Overview page to viewport a high-level view of your Clusters, usage, and alerts.

Time range and filtering

Use the time range tool and filtering to narrow the scope to what you're looking for.

  • Adjust the time range to view historical data - try setting it to "Last 24 hours" for better visibility.
  • Use the cluster filter to focus on a specific cluster or view all clusters.

Search

Find and navigate to any Kubernetes object using search.

  • Use the search bar to find a specific workload. Try searching for "nginx" or any workload name you know exists.
  • Click on the first search result to open that workload's detail page.
  • 💡 Navigation Tip: Notice the breadcrumb navigation at the top. You can click any level to jump back up the hierarchy.
  • On the workload detail page, look for links to view the pods in this workload.
  • Click on a pod name to drill down to the pod level and see individual pod metrics.

Understanding the Kubernetes Hierarchy

Kubernetes Monitoring reflects the natural structure of Kubernetes:

  • Clusters → contain multiple nodes and namespaces
  • Namespaces → organize workloads within a cluster
  • Workloads → deployments, daemonsets, statefulsets, etc.
  • Pods → individual instances of your applications
  • Containers → the actual processes running your code

Step 4: Analyze Performance with Specialized Tabs

Each detail page offers specialized views for different aspects of performance. Let's explore the CPU and Memory tabs.

  • Click the CPU tab to see detailed CPU analysis.
  • 📊 What you're seeing:
    • Usage vs. Capacity: How much CPU is being used compared to what's available
    • Usage vs. Requests: How actual usage compares to what was requested
    • Throttling: When containers are being limited due to resource constraints
  • Switch to the Memory tab to see memory utilization patterns.
  • Examine the memory usage graphs. Look for any containers approaching their memory limits.
  • Click the Network tab to understand bandwidth usage and potential network saturation.

Step 5: Investigate Alerts and Issues

Learn how to quickly identify and investigate issues using the alerts and troubleshooting features.

  • Navigate to the Alerts page to see all Kubernetes-related alerts.
  • Sort alerts by "Firing Since" to prioritize the most critical or longest-running issues.
  • Click on a pod or container name in an alert to jump directly to its detail page.
  • 🔧 Troubleshooting Tools Available:
    • Run Sift Investigation: Automated diagnostic checks
    • Explore buttons: Deep-dive into raw metrics
    • RCA Workbench: Root cause analysis (requires Asserts)
    • Logs & Events tab: View related log entries
  • If available, try clicking "Run Sift Investigation" to start an automated diagnostic.

Step 6: Optimize Costs and Efficiency

Use the cost analysis and efficiency tools to optimize your Kubernetes resource usage and spending.

  • Navigate to the Cost page to analyze your Kubernetes spending.
  • Click the Savings tab to see optimization recommendations.
  • 💰 Cost Optimization Insights:
    • Stranded Resources: Unused CPU/memory you're paying for
    • Over-provisioned Workloads: Resources that could be rightsized
    • Cost per Namespace: Understanding which teams/apps cost the most
  • Navigate to the Efficiency page to see resource utilization patterns.
  • Click on a workload in the efficiency table to see detailed resource usage analysis.

Step 7: Use Machine Learning Predictions

Leverage Grafana Cloud's ML capabilities to predict future resource usage and detect anomalies.

  • 🤖 Prerequisites: Machine Learning features require the ML plugin to be enabled on your Grafana Cloud stack.
  • Look for and click a "Predict CPU" button on any detail page to see future CPU usage predictions.
  • Try the "Predict Mem Usage" button to forecast memory requirements.
  • 📈 Using Predictions:
    • Capacity Planning: Ensure resources will be available during predicted spikes
    • Cost Optimization: Right-size resources based on predicted usage patterns
    • Scaling Decisions: Know when to scale up or down proactively
  • On multi-pod workloads, look for "outliers by CPU" links to identify problematic pods.
