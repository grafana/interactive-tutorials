üìö Select Your Data Source

Before we can query logs, we need to select a Loki data source. The data source picker allows you to choose which Loki instance to query against.

  • The data source picker allows you to select which Loki data source to query. In Grafana, you can have multiple Loki instances configured (e.g., production logs, development logs, different regions). The Loki Demo Kit (loki_nginx) data source contains sample logs that we'll use for this tutorial. Selecting this data source will connect you to the specific Loki instance where you can explore and query logs. Click on the data source picker, then select the Loki Demo Kit (loki_nginx) data source.

‚úçÔ∏è Write Your First LogQL Query

Now that we have our data source selected and Code mode enabled, let's write our first LogQL query! Every LogQL query starts with a label selector enclosed in curly braces.

  • The Code mode allows you to write LogQL queries directly as text, giving you full control over the query syntax. This is perfect for learning LogQL's powerful query language. Switch to Code mode by clicking the raw query toggle to write LogQL directly.
  • Always start a LogQL query with a label selector. They always appear within curly braces. Write your first query by typing 
    {service_name="nginx_access_log"}
    into the query field. This query selects all log lines from the Nginx access log service. Label selectors use the format {label_name="value"} to filter logs by specific attributes. Add as many label selectors as you can to filter your results down faster!
  • The Run query button executes your LogQL query and displays the results. This is how you see the actual log data that matches your query criteria. Click the Run query button to execute your LogQL query and see the results.
