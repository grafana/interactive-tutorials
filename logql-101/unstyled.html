Welcome to LogQL 101! 📝

Welcome to your hands-on introduction to LogQL - Grafana Loki's powerful query language for logs! This interactive guide will teach you how to search, filter, and analyze logs like a pro.

LogQL is inspired by PromQL (Prometheus Query Language) and provides a simple yet powerful way to query and aggregate log data. Let's get started!

🎯 What You'll Learn

  • 🔍 Basic log queries - Search and filter logs
  • 🏷️ Label selectors - Filter by metadata
  • 📊 Log aggregations - Count and summarize log data
  • ⚡ Advanced patterns - Parse and extract values from logs

🚀 Let's Get Started

In this tutorial, you'll learn how to craft powerful LogQL queries to extract insights from your log data. We'll start with simple queries and progressively build up to more advanced use cases.

📚 Select Your Data Source

Before we can query logs, we need to select a Loki data source. The data source picker allows you to choose which Loki instance to query against.

  • The data source picker allows you to select which Loki data source to query. In Grafana, you can have multiple Loki instances configured (e.g., production logs, development logs, different regions). The App o11y frontend data source is a Loki instance that contains logs from the application observability frontend services. Selecting this data source will connect you to the specific Loki instance where you can explore and query logs from the frontend application. Click on the data source picker, then select the Loki Demo Kit (loki_nginx) data source.
  • Switch to Code mode by clicking the raw query toggle to write LogQL directly.
  • Start off your query by typing the opening brace (`{`). 
    {
  • Enter this LogQL query: 
    {container="alloy"}
