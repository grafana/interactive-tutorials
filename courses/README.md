# Courses

This directory holds the **course catalog** served to Pathfinder via the
interactive-learning CDN. The Pathfinder plugin fetches one of the two
platform index files on startup and renders cards from it; on fetch
failure it falls back to a bundled offline course.

Final CDN URLs:

- `https://interactive-learning.grafana.net/courses/oss.json`
- `https://interactive-learning.grafana.net/courses/cloud.json`

## Structure

```
courses/
  oss.json                              ← generated by CI (DO NOT hand-edit)
  cloud.json                            ← generated by CI (DO NOT hand-edit)
  badges.json                           ← source-of-truth, all badges
  <course-id>/
    course.json                         ← source-of-truth, per course
```

Per-course files and the single `badges.json` are the **authoritative
source**. The deploy workflow aggregates them into `oss.json` and
`cloud.json` using `pathfinder-cli build-courses`. Edit the per-course
files / `badges.json`; let CI rebuild the indexes.

## Adding a course

1. Create `courses/<id>/course.json` using the shape below.
2. If the course awards a new badge, append it to `courses/badges.json`.
3. Open a PR — CI validates and rebuilds the indexes on merge.

### `course.json`

```jsonc
{
  "schemaVersion": "1.0.0",
  "id": "getting-started",
  "title": "Getting started with Grafana",
  "description": "Build your first dashboard.",
  // Each entry is either a bare package ID (resolved via the plugin's
  // package-engine) or an absolute https:// URL (rendered as an external-
  // link guide). Pathfinder distinguishes via `entry.startsWith('http')`.
  "guides": [
    "welcome-to-grafana",
    "first-dashboard",
    "https://grafana.com/docs/learning-paths/getting-started/"
  ],
  "guideMetadata": {
    "welcome-to-grafana": { "title": "Welcome", "estimatedMinutes": 5 },
    "first-dashboard": { "title": "First dashboard", "estimatedMinutes": 10 }
  },
  "badgeId": "grafana-fundamentals",
  "targetPlatform": "oss",     // "oss" | "cloud" — omit for both
  "estimatedMinutes": 25,
  "icon": "grafana"
}
```

### `badges.json`

```jsonc
{
  "schemaVersion": "1.0.0",
  "badges": [
    {
      "id": "grafana-fundamentals",
      "title": "Grafana Fundamentals",
      "description": "Complete the \"Getting started with Grafana\" course",
      "icon": "grafana",
      "emoji": "🎓",                       // optional; preferred over Grafana icon when set
      "trigger": { "type": "path-completed", "pathId": "getting-started" }
    }
    // … one entry per badge
  ]
}
```

Trigger types:

- `{ "type": "guide-completed", "guideId": "<id>" }` — `guideId` optional (any guide if omitted)
- `{ "type": "path-completed", "pathId": "<course-id>" }`
- `{ "type": "streak", "days": <n> }`

## Schema versioning

`schemaVersion` is bumped only on **breaking** changes. Additive fields
are forward-compatible because plugin-side parsing uses Zod `.loose()`.

The authoritative schema lives in the plugin repo:
[`grafana-pathfinder-app/src/types/courses.schema.ts`](https://github.com/grafana/grafana-pathfinder-app/blob/main/src/types/courses.schema.ts).
Breaking schema changes must land in `interactive-tutorials` **before**
the plugin release that requires the new shape, or the CDN will start
serving content that the plugin rejects (and users will fall back to the
bundled offline course).

## Local validation

```bash
# In a checkout of grafana-pathfinder-app
npm run build:cli
node dist/cli/cli/index.js build-courses /path/to/interactive-tutorials/courses --validate-only
```

This runs the same Zod schema CI uses, plus integrity checks (every
course's `badgeId` resolves; every `path-completed` badge points to a
real course).
