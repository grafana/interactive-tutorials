How To Set Up Your First Synthetic Monitoring Check

In this interactive tutorial, you'll learn how to set up and run an HTTP check in Grafana Cloud Synthetic Monitoring. By the end of this tutorial, you will:

  • Know how to create and configure HTTP checks
  • Understand how Synthetic Monitoring helps you track uptime and performance
  • Be able to set up alerts to stay informed about your service status

Section 1: Navigate to Testing & Synthetics Checks

Let's start by navigating to the Testing & Synthetics section. This is where you'll create and manage your synthetic checks.

  • First let's open the main menu and navigate to Testing & synthetics.
  • Next, click Synthetics.

Section 2: Create and Configure Check

Here, you will create a new check and configure its parameters.

  • Click Add NewCheck.
  • Choose API Endpoint as the check type.
  • Enter Grafana: HTTPCheck in Job Name Field.
  • Choose HTTP as the request type.
  • Set Method: GET.
  • Set URL: https://grafana.com.
  • Click Define Uptime.

Section 3: Define Labels

Next, you'll define uptime settings and add labels for the check.

  • Click the Add label button.
  • Add name: name: grafana.
  • Add value: value: test.
  • Click the Execution button.

Section 4: Configure Execution

Finally, you'll configure probe locations and frequency for the check.

  • Choose Public probes and select the Americans
  • Choose Frequency and select 1m
  • Click Publish full set of metrics
  • Click the Alerting button.

Section 5: Configure Alerting

Finally, you'll configure alerting for the check.

  • Choose Per-check alerts to configure alerts
  • Click the checkbox to enable the failed check alert
  • Click the checkbox to enable the TLS Certificate alert
  • Click the checkbox to enable the Latency alert
  • Click Save to create the 3 alerts
