How to set up your first Synthetic Monitoring check

In this interactive guide you'll learn how to set up and run an HTTP check in Grafana Cloud Synthetic Monitoring. You will:

  • Know how to create and configure HTTP checks
  • Understand how Synthetic Monitoring helps you track uptime and performance
  • Learn how to set up alerts to stay informed about your service status

Section 1: Navigate to adding a new HTTP check

Let's start by navigating to the Testing & Synthetics section. This is where you'll create and manage your synthetic checks.

  • In the main menu and navigate to Testing & synthetics.
  • Next, click Synthetics.
  • Click Create new check.
  • Choose API Endpoint as the check type.

Section 2: Configure the target of the Synthetic Monitoring check

In this section you will configure the Synthetic Monitoring check's parameters.

  • All your logs and metrics generated by this check will contain a job label with this value. Enter Grafana: HTTP Check in the Job Name field.
  • They will also contain an instance label with this value. Set URL: https://grafana.com.

Section 3: Review Optional Parameters for the check

In this section you can define uptime parameters for the check.

  • This is the form stepper for configuring your check. You can freely move between sections whenever you want. Click Uptime to move to the next section of the check configuration.
  • Status codes in the 2xx range are the default, so we can skip this step. Set Valid status codes if required.
  • Move to the Labels section.
  • This is optional; but if you have many checks, this can help you organize them and route your alerts to the right team. Common labels include 'environment', 'service', and 'team' but you are free to use any label you want. Set label names and values if required.

Section 4: Execution

Setup probe location and frequency for the check.

  • Move to the Execution section.
  • There are multiple probe locations to choose from. It is good practice to run checks from multiple locations to ensure the check is reliable. Select a probe location to run the check from.
  • The default probe frequency is 1 minute which is suitable for most checks. Less frequent means fewer executions, but longer lag times before an alert notifying you of an issue. Set the desired probe frequency.

Section 5: Configure Alerting

Configure alerting for the check.

Finally, you'll configure alerting for the check.

  • Move to the Alerting section.
  • The failed check alert will be triggered if the check fails more than the configured number of times. Click the checkbox to enable the failed check alert
  • You can click Test to see if the check is working as expected.
  • Click Save to create the check.

Congratulations!

You have completed the HTTP check tutorial in Grafana Cloud Synthetic Monitoring. You have successfully:

  • Created and configured an HTTP check to monitor uptime and performance
  • Added a probe location and frequency to the check
  • Set up an alert to stay informed about service issues
