# Shared snippets

Reusable JSON blocks for common Grafana UI patterns. Snippets serve two consumers:

1. **Guide authors** can copy-and-adapt the JSON into their own guide when they want a self-contained, locally-modifiable version.
2. **The Pathfinder app** can reference a snippet by ID via a `snippet-ref` block, so a guide always renders the latest published version of the snippet — a single edit here propagates to every consumer.

## Layout

```
shared/snippets/
  index.json           # catalog, generated by scripts/build-snippets-index.py
  <id>.json            # one snippet body per file
```

Each body is a JSON object with these fields:

| Field           | Required | Notes                                                                                          |
| --------------- | -------- | ---------------------------------------------------------------------------------------------- |
| `id`            | yes      | Kebab-case, must match the filename (`time-picker.json` → `"id": "time-picker"`)               |
| `title`         | yes      | Short human-readable name, shown in the picker                                                 |
| `description`   | yes      | One-sentence summary shown in the picker — without this, browsing snippets is unusable         |
| `category`      | no       | Optional grouping label                                                                        |
| `tags`          | no       | Optional array of search keywords                                                              |
| `schemaVersion` | no       | Defaults to the current snippet schema version when absent                                     |
| `blocks`        | yes      | Array of JSON guide blocks. Cannot contain `snippet-ref` — snippets cannot reference snippets. |

## Adding or editing a snippet

1. Create or edit `shared/snippets/<id>.json`. Include all required fields.
2. Regenerate the index from the bodies:
   ```bash
   python3 scripts/build-snippets-index.py
   ```
3. Commit both the body and the regenerated `index.json`. CI checks that they stay in sync.

The body is the source of truth — `index.json` is derived. Never hand-edit `index.json`.
